Workflow Cheatsheet

Files are buffers. Bufferline shows files. Panes are layout. Tabs are workspaces.
Leader: Space
Rescue: Space ?
Daily loop: Space f H/L Space tt

Core Actions

Space fFind file on disk
Space bFind open file
H / LPrev / next open file
Space /Search text in project
Space eExplorer
Space ?Find keymaps

Picker Mode

EnterOpen selection
Ctrl-vOpen in vertical pane
Ctrl-xOpen in horizontal pane
Ctrl-tOpen in Vim tab
Alt-dDelete buffer in Space b
EscClose picker
Inside Telescope, use picker keys — not Space mappings.

Panes + Tabs

Space v/hVertical / horizontal pane
Space xClose pane
Ctrl-h/j/k/lMove panes
ArrowsResize pane
gt / gTNext / prev workspace tab
:tabnewNew workspace

Terminal + Code

Space ttTerminal below
Space tfFloating terminal
Esc / jkLeave terminal mode
gd / KDefinition / docs
Space r/a/FRename / action / format
gcc / gcComment line / selection

Opening Workflows

Open a file
Space f -> type name -> Enter
Open beside current
Space f -> select file -> Ctrl-v vertical or Ctrl-x horizontal
Open test beside code
Space f implementation -> Space f test -> Ctrl-v -> Ctrl-h/l
Search then inspect
Space / query -> Enter result -> gd for definition if needed
Open separate context
Space f -> selected file -> Ctrl-t; move workspaces with gt/gT

Buffer Workflows

Nearby open file
H/L until it appears in current pane
Far open file
Space b -> type part of filename -> Enter
Prune open files
Space b -> highlight file -> Alt-d; repeat, then Esc
Close current file
Space bd closes the buffer; use Space x only to close the pane
Keep important file
Space bp pin it -> Space bo close other buffers

Terminal / Test Workflows

Run tests once
Space tt -> run test command -> Esc -> Ctrl-k back to code
Keep server running
Space t1 for server -> Space t2 for tests/logs
Quick shell command
Space tf -> run command -> close or Esc back out
Terminal got focus
Esc or jk leaves terminal mode; Ctrl-h/j/k/l moves panes

Editing / Debugging Workflows

Follow code path
gd definition -> H/L switch files -> Ctrl-o jump back if needed
Refactor name
Space r rename symbol -> Space F format
Fix warning
]d next diagnostic -> Space a code action -> Space F
Comment experiment
gcc current line; select lines then gc for block

Project / Git Workflows

Review changed code
Space / search symbol -> Space gp preview current git hunk
Compare implementation
Space f target file -> Ctrl-v related file -> move with Ctrl-h/l
Read docs / README
Space f README or note -> Space um toggle Markdown render
Python environment
Space V select venv before relying on imports, hover, rename, or diagnostics

Housekeeping / Recovery

Lost in mappings
Space ? search keymaps; press Space and pause for which-key
Wrong mode
Esc first, then leader mapping like Space v or Space f
Layout too messy
Space = equalize panes; Space x close extra pane; Space bo prune files
Save / quit / clipboard
Space w save; Space q quit; Space y/p clipboard
